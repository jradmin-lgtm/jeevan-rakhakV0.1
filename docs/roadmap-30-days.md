# 30-Day Delivery Roadmap

## Week 1 - Foundation
- Day 1: confirm assumptions and scope lock.
- Day 2: finalize schema and API contracts.
- Day 3: scaffold monorepo and shared packages.
- Day 4: setup API auth and booking stubs.
- Day 5: setup CI pipeline and environment templates.
- Day 6: integrate base logging and error handling.
- Day 7: architecture review and backlog reprioritization.

## Week 2 - User App Core
- Day 8: OTP login flow screens and API wiring.
- Day 9: home + location permission and pickup selection.
- Day 10: emergency category and booking confirmation.
- Day 11: ETA screen and tracking state handling.
- Day 12: payment method and receipt flow.
- Day 13: booking history and ratings.
- Day 14: QA cycle for user happy path.

## Week 3 - Driver + Realtime
- Day 15: driver login and onboarding states.
- Day 16: KYC upload and approval status.
- Day 17: availability and dispatch request handling.
- Day 18: trip lifecycle updates (accept to complete).
- Day 19: location update frequency controls.
- Day 20: admin live map integration.
- Day 21: end-to-end dispatch simulation tests.

## Week 4 - Admin + Launch Readiness
- Day 22: dashboard KPIs and booking controls.
- Day 23: driver approval and incident handling.
- Day 24: complaints and refund workflow.
- Day 25: pricing configuration and guardrails.
- Day 26: security hardening and abuse checks.
- Day 27: UAT and bug bash.
- Day 28: performance and reliability fixes.
- Day 29: runbook and on-call drill.
- Day 30: staged launch and pilot monitoring.
